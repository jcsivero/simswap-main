Instrucciones de instalación para los programas de Visual Studio

Esta sección presenta información relacionada con la instalación de los programas y archivos incluidos con Visual Studio 6.0.

Instalar sobre una instalación anterior de una Beta, una versión preliminar o una Vista previa técnica de Visual Studio

No instale la versión final de Visual Studio 6.0 en equipos en los que haya instaladas versiones preliminares, vistas previas técnicas o betas de algún producto de Visual Studio. El destino de los archivos y la configuración de registro han cambiado durante el desarrollo. Si instala la versión final en equipos con una Vista previa de tecnología, una versión preliminar o una beta, puede que no se instale algún componente final, mientras que la versión beta anterior permanecerá en el sistema. Como resultado, el producto puede resultar inestable o, aún peor, el sistema puede resultar inestable.

Vuelva a formatear el disco duro y a instalar el sistema operativo antes de volver a utilizar en un entorno de producción un equipo que se haya utilizado como plataforma de pruebas beta.

Instalar múltiples versiones de productos de Visual Studio 6.0 en el mismo equipo

Los productos de Visual Studio con la misma versión y el mismo idioma, Visual C++ y Visual Basic versión 6.0 en inglés, por ejemplo, pueden instalarse de forma independiente en la misma estación de trabajo. Microsoft permite estas instalaciones. El Asistente para la instalación de Visual Studio 6.0 detecta si hay instaladas otras versiones de la línea de productos 6.0 en la estación de trabajo de un programador.

En general, no se pueden mezclar versiones de diferentes idiomas, de diferente compilación o de distinto nivel de producto en la misma estación de trabajo. Esto significa que la instalación puede fallar, uno o más de los productos de Visual Studio 6.0 podrían no funcionar aunque la instalación se realice correctamente o, en el peor de los casos, no podría desinstalar ningún producto por completo.

Si el asistente muestra una advertencia durante la instalación, lo más seguro es desinstalar el producto de Visual Studio 6.0 antes de continuar. El esquema de detección comprueba los siguientes escenarios:

  • Usuarios que instalan versiones en diferentes idiomas del mismo producto. Por ejemplo, Visual Basic en japonés y Visual Basic en inglés. Se admiten la instalación de ciertas combinaciones.

    • Producto Instalación admitida
    Visual Basic Instalar en dos ubicaciones diferentes
    Visual C++ No se pueden instalar varias versiones.
    Visual FoxPro Instalar en dos ubicaciones diferentes
    Visual InterDev Instalar en la misma ubicación. Debe volver a instalar después de desinstalar una versión.
    Visual J++ Instalar en la misma ubicación. Debe volver a instalar después de desinstalar una versión.
    Visual SourceSafe Instalar en la misma ubicación. Debe volver a instalar después de desinstalar una versión.

    Nota   El idioma de interfaz de usuario seleccionado al ejecutar SourceSafe dependerá de la configuración de idioma del sistema. Si el idioma no está instalado en SourceSafe, utilizará inglés.
    MSDN Instalar en el directorio predeterminado.

  • Usuarios que instalan diferentes niveles de productos de Visual Studio 6.0. Por ejemplo, Visual J++ Edición estándar y Visual InterDev Edición profesional. No se pueden instalar diferentes niveles del mismo producto en la misma estación de trabajo.

  • Usuarios que instalan versiones diferentes. Se puede actualizar a una versión posterior de Visual Studio sobre una versión anterior. Sin embargo, no es posible instalar versiones anteriores sobre versiones posteriores.

  • Usuarios que instalan versiones independientes de los productos de Visual Studio 6.0 sobre las versiones instaladas por Visual Studio 6.0 Edición profesional o Edición empresarial. Por ejemplo, Visual FoxPro 6.0 sobre Visual Studio Edición profesional 6.0. Es posible instalar una versión independiente del mismo idioma (inglés) de los productos de Visual Studio 6.0. Es seguro instalar Visual FoxPro 6.0 (inglés) desde la versión independiente en una estación de trabajo en la que ya esté instalado Visual Basic 6.0 desde Visual Studio Edición profesional.

Advertencia al realizar la instalación en equipos con arranque dual

Si dispone de un equipo con arranque dual y desea instalar y ejecutar los programas de Visual Studio con ambos sistemas operativos, deberá instalar los productos en directorios diferentes.

Si se instala un producto de Visual Studio, como Visual C++ o Visual Basic, varias veces en el mismo directorio, se puede producir un comportamiento errático. En ocasiones se instala un producto para varios sistemas operativos en la misma ubicación para ahorrar espacio en el disco, pero producirá problemas y no se admite.

Por ejemplo, si realiza la instalación en el mismo directorio, no podrá desinstalar el producto por completo desde ambos sistemas operativos. Si se desinstala en un sistema se puede producir un comportamiento indefinido, en el que se puede incluir la eliminación de archivos de sistema del otro sistema operativo.

Instalar Visual Studio desde una unidad de CD compartida

Si instala Visual Studio 6.0 desde una unidad de CD compartida, utilice un nombre de recurso compartido de menos de nueve caracteres y sin espacios incrustados.

Deje que termine el programa de instalación de Microsoft Internet Explorer antes de realizar la instalación de Visual Studio

Durante la instalación de Visual Studio, el Asistente para la instalación instalará Internet Explorer 4.01 Service Pack (SP) 1. Casi al final de la instalación de Internet Explorer, el Asistente para la instalación de Internet Explorer reiniciará su equipo. El Asistente para la instalación de Internet Explorer volverá a ejecutarse automáticamente y seguirá copiando archivos y configurando su sistema.

En algunos equipos, el Asistente para la instalación de Visual Studio puede ejecutarse también mientras el Asistente para la instalación de Internet Explorer finaliza esta tarea. Debe dejar que el Asistente para la instalación de Internet Explorer finalice este proceso antes de continuar con la instalación de Visual Studio.

En concreto, es esencial que no continúe con la instalación hasta llegar al punto en el que haya que volver a iniciar el sistema. Si reinicia el equipo durante la fase final de configuración de Internet Explorer, dañará la instalación de Visual Studio. El Asistente para la instalación de Internet Explorer muestra un indicador de progreso mientras está en su fase final. Cuando desaparezca este indicador ya será seguro continuar con el Asistente para la instalación de Visual Studio.

Cuando el programa de instalación de Microsoft Internet Explorer instala versiones antiguas de los archivos MDAC

Los archivos de Microsoft Data Access (MDAC) instalados por Microsoft Internet Explorer 4.0 pueden ser anteriores a los archivos MDAC instalados por los productos de Visual Studio 6.0. Si instala Internet Explorer en un equipo que ya tenga Visual Studio 6.0 y aparece un mensaje que le pida que confirme si desea instalar las versiones más antiguas de los archivos MDAC, elija la opción que le permite mantener los archivos que tiene instalados actualmente en su equipo.

Cuando el Asistente para la instalación busca el disco del Asistente para la publicación en Web

Al instalar Visual Studio 6.0 o Visual J++ Technology Preview 2, si aparece un mensaje de error pidiéndole que introduzca el disco del Asistente para la publicación en Web, cierre el cuadro de diálogo y continúe con la instalación.

Ni Visual Studio ni Visual J++ incluyen un Asistente para la publicación en Web en un disco independiente. Es posible que se presente este mensaje cuando los usuarios instalan Visual Studio o Visual J++ en equipos que tengan directorios Temp configurados en unidades extraíbles, como unidades Iomega Jaz.

Cuando termine la instalación de Visual Studio o Visual J++, si desea instalar el Asistente para la publicación en Web deberá realizar los siguientes pasos:

  1. Configure la variable de entorno Temp para un directorio de una unidad fija, como su disco duro.
  2. Ejecute wpie15.exe desde [unidad del CD:] \setup del disco compacto.
  3. Una vez que haya finalizado la instalación del Asistente para la publicación en Web, vuelva a configurar su variable de entorno Temp tal y como estaba inicialmente, si es necesario.

Instalar herramientas de cliente y componentes de servidor después de instalar los productos de Visual Studio 6.0

El proceso de instalación está formado por varios componentes. Estos componentes no se ejecutan desde Agregar o quitar Programas en el Panel de control. Cuando utilice Agregar o quitar programas, sólo verá la parte del producto que corresponde a la instalación del producto.

Para instalar cualquiera de las siguientes herramientas deberá ejecutar la instalación completa.

  • Application Performance Explorer
  • Data Access Components 2.0
  • FrontPage 98
  • Extensiones de servidor de FrontPage 98
  • InstallShield
  • NT Option Pack para Windows 95
  • Ejemplos de Visual FoxPro Server
  • Componentes de Visual InterDev Server
  • Visual SourceSafe Server
  • Visual Studio Analyzer
  • MSDN

Para instalar cualquier herramienta de esta lista después de instalar los productos de Visual Studio 6.0, ejecute instalar.exe desde el disco 1. Así se ejecutará la instalación completa en "modo de mantenimiento". Dispondrá de todas las opciones que tenía al instalar por primera vez el producto de Visual Studio 6.0, pero no tendrá que pasar por todos los cuadros de diálogo preliminares.

El mensaje "El programa de instalación ya se está ejecutando" aparece durante la instalación de los componentes de servidor

Al instalar los componentes de servidor de Visual Studio 6.0 en un equipo con Windows NT Server mediante el Asistente para la instalación, puede encontrarse con el mensaje "Ya se está ejecutando otra copia de la Instalación. Termine la otra instalación o reinicie Windows".

Esta situación suele producirse en equipos lentos en los que el cronómetro del asistente salta mientras espera a que termine la instalación de ACME y el Asistente inicia la siguiente instalación.

Haga clic en Aceptar en el cuadro del mensaje. Esta acción indicará al asistente que debe detener uno de los procesos en conflicto y continuar con el resto de las instalaciones. La página de resumen que se genera automáticamente al finalizar la instalación mostrará una lista con los componentes que no se instalaron completamente, si hay alguno.

Para instalar el componente que ha fallado, ejecute de nuevo el asistente para la instalación.

Instalar componentes de servidor en Windows 95

Se recomienda que instale todas las aplicaciones de servidor en equipos que ejecuten Windows NT.

Las ediciones empresarial y profesional de los productos de Visual Studio 6.0 incluyen varias aplicaciones de servidor de Microsoft diseñadas específicamente para el programador que trabaja con Visual Studio. La mayoría de las aplicaciones pueden ser instaladas en Windows 95 o Windows NT. Le recomendamos que instale todas las aplicaciones de servidor en Windows NT, Workstation o Server.

Si decide instalar los servidores en un equipo con Windows 95, el orden de instalación será importante. En Windows 95, algunas aplicaciones deben instalarse antes que otras para que la instalación se realice correctamente y el producto funcione. Instale los componentes en este orden:

  1. NT Option Pack
  2. Extensiones de servidor de FrontPage 98
  3. Data Access Components 2.0
  4. Componentes de Visual InterDev Server
  5. Application Performance Explorer
  6. Visual Studio Analyzer
  • (En cualquier orden) Visual SourceSafe Server
  • (En cualquier orden) Ejemplos de Visual FoxPro Server
El Asistente para la instalación presenta estas opciones en el mismo orden. De nuevo, el orden sólo es importante en Windows 95.

Para obtener información acerca de otros problemas relacionados con Windows 95, consulte Problemas específicos de equipos con Windows 95

Problemas de compatibilidad con SQL Server 6.5 y Windows NT Option Pack 4.0 conocidos

Si instala SQL Server 6.5 en un equipo con Windows NT Option Pack 4.0 instalado, puede que se sobrescriban algunos de los controles en tiempo de diseño (DTC).

Si esto sucede, aparecerá un mensaje de error cuando inicie Microsoft Transaction Server (MTS). El mensaje le indicará que debe volver a instalar MTS.

Para evitar esta situación, instale NT Option Pack después de instalar SQL Server.

Configurar para la depuración

La información de esta sección trata problemas relacionados con la depuración y la configuración de equipos para servicios de depuración.

Para depurar ASP, instale la revisión de IIS después de instalar BackOffice

Internet Information Server (IIS) tiene una revisión que soluciona algunos problemas relacionados con la depuración de páginas Active Server (ASP) desde Visual InterDev. Esta revisión también soluciona un problema de Windows 95/98, por el que los cambios en Global.asa no quedan reflejados en su aplicación (como agregar, cambiar o quitar conexiones de datos) hasta que detiene y vuelve a iniciar su servidor Web.

Para asegurarse de que la depuración ASP funciona correctamente o para solucionar el problema con Windows 95/98, aplique la revisión después de que termine el Asistente para la instalación de BackOffice.

Para aplicar la revisión de IIS en ordenadores con Windows NT

  1. Ejecute \IIS\WinNT\ASP2FIX1.exe desde el disco de instalación.

    2. Producto Ubicación en disco
    Visual Studio, Edición profesional CD 2
    Visual Studio, Edición empresarial CD 3
    Visual InterDev CD 1

  2. Reinicie el sistema cuando se le indique.

Para aplicar la revisión de IIS en equipos con Windows 9x

  1. Detenga el servidor Web desde el indicador de línea de comandos: escriba pws/stop y presione ENTRAR.

  2. En el directorio \<windows>\system\inetsrv, cambie el nombre de Asp.dll por Asp.old

  3. Desde el directorio \IIS\Win9x\ del disco de instalación, copie Asp.dll y péguelo en \<windows>\system\inetsrv en su equipo.

    4. Producto Ubicación en disco
    Visual Studio, Edición profesional CD 2
    Visual Studio, Edición empresarial CD 3
    Visual InterDev CD 1

  4. Vuelva al indicador de línea de comandos y reinicie el servidor Web: escriba pws/start y presione ENTRAR.

Utilizar los servicios de depuración de SQL en un equipo con SQL Server 6.5 y Visual C++ 5.0 o posterior

Para utilizar los servicios de depuración de SQL en un equipo con SQL Server 6.5 y Visual C++ 5.0 o posterior, asegúrese de que las propiedades de configuración de la Automatización remota (AR) de su equipo son correctas.

Para comprobar las propiedades de configuración de AR

  1. Ejecute dcomcnfg.exe.
  2. Seleccione Remote Automation CLSID_StubMarhsaler.
  3. Haga clic en Propiedades.
  4. En la ficha Identidad, seleccione Usuario interactivo.

Además, sus escenarios de depuración de SQL podrían no funcionar si ejecuta SQL Server como un servicio bajo la cuenta Local System. Debe seleccionar Esta cuenta e introducir una cuenta y una contraseña válidas.

Problemas específicos de equipos con Windows 9x

Esta sección muestra problemas identificados al instalar Visual Studio 6.0 en equipos con Windows 95 o Windows 98.

Tamaño del Registro de Windows 95: el límite de 64 KB puede hacer que aparezca una advertencia durante la instalación de Visual Studio

La limitación a 64 KB del tamaño de la clave SharedDLL en el registro del sistema de Windows 95 puede generar una advertencia al instalar Visual Studio 6.0.

Antes de seguir con la instalación, debería acortar la ruta del directorio en el que piensa instalar Visual Studio 6.0, reduciendo así la cantidad de espacio necesario en el registro. Además, puede desinstalar programas que ya no utilice mediante Agregar o quitar programas del Panel de control. Se eliminarán las claves del Registro utilizadas exclusivamente por estos programas.

Para ahorrar más espacio de Registro utilizado por los programas que desea conservar, desinstale y vuelva a instalar estos programas para acortar las rutas de instalación.

Considere la posibilidad de usar la utilidad RegClean, diseñada para limpiar las entradas innecesarias del Registro. Para obtener más información y un vínculo de descarga, vaya a http://support.microsoft.com/support/kb/articles/q147/7/69.asp

Nota   En equipos con Windows NT Workstation y Windows NT Server, el programa de instalación de Visual Studio cambiará automáticamente el tamaño del Registro.

Deshabilitar el inicio de sesión automático antes de ejecutar el programa de instalación de Visual Studio 6.0

El programa de instalación de Visual Studio puede fallar en equipos con Windows 95 configurados con inicio de sesión automático.

Antes de instalar Visual Studio 6.0 o cualquiera de sus productos en un equipo con Windows 95 que tenga habilitado el inicio de sesión automático, desactive el inicio de sesión automático siguiendo las instrucciones que se encuentran en http://support.microsoft.com/support/kb/articles/q141/8/58.asp.

Note   Este artículo contiene varios pasos adicionales que pueden no ser relevantes en su sistema.

Si ha instalado productos de Visual Studio en un equipo con Windows 95 configurado con el inicio de sesión automático y la instalación ha fallado, repita los pasos de la instalación con el inicio de sesión automático deshabilitado.

Puede que aparezca un cuadro de diálogo de conexión a Internet en el inicio de Windows o de otras aplicaciones

Si se permiten conexiones remotas en Windows 95 o Windows 98, puede que el sistema intente inicializar una conexión a Internet en el inicio de Windows o en el inicio de muchas aplicaciones. Este comportamiento se llama a veces "marcado automático" o "conexión automática". Puede desactivar este comportamiento si establece la clave del Registro EnableRemoteConnect a "N".

Para ello, cree un archivo de texto "DisbleAutoConnect.reg" con las tres líneas siguientes:

REGEDIT4
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\OLE]
"EnableRemoteConnect"="N"

Cambiar esta configuración para deshabilitar conexiones remotas no debería evitar ninguna de sus actividades normales en Internet; esta configuración es el valor predeterminado para la mayoría de los sistemas. Sin embargo, para algunas características de DCOM es necesario habilitar las conexiones remotas (vea http://support.microsoft.com/support/kb/articles/Q177/3/94.asp).

Si necesita a veces tener habilitadas las conexiones remotas, cree otro archivo REG, "EnableRemoteConnect.reg", con las tres líneas siguientes:

REGEDIT4
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\OLE]
"EnableRemoteConnect"="Y"

Ejecute "EnableAutoConnect.reg" para establecer esta configuración y, a continuación, ejecute "DisbleAutoConnect.reg" para deshabilitarla cuando ya no sea necesaria la conexión automática.

Puede que el Asistente para instalación no se reinicie después de que se haya instalado PWS en equipos con Windows 95

Si elige instalar Personal Web Server (PWS) en un equipo con Windows 95, puede que el Asistente para la instalación no reanude correctamente su ejecución cuando se reinicie el equipo. Para continuar, vuelva a ejecutar Instalar.exe desde el primer disco del conjunto de CD de Visual Studio 6.0. El Asistente para la instalación reanudará automáticamente la instalación en el punto correcto.

Cómo instalar las Extensiones de Microsoft FrontPage 98 en equipos con Windows 95

Cuando el Asistente para la instalación ejecuta otras instalaciones de clientes o servidores, el Asistente se vuelve invisible y espera hasta que el componente esté instalado. Después, el Asistente vuelve a aparecer y permite al usuario instalar otro componente.

Cuando instale las Extensiones de Microsoft FrontPage 98 en algunos equipos con Windows 95, puede que el Asistente para la instalación permanezca oculto durante varios minutos después de que se haya completado la instalación de FrontPage.

Mientras el Asistente esté oculto no podrá ejecutar de nuevo el programa de instalación. No se iniciará un segundo Asistente para la instalación cuando el programa detecte que ya hay uno en ejecución.

Notas acerca del efecto sobre otros programas instalados

En esta sección se muestran problemas de instalación que pueden afectar a otros programas ya instalados en su equipo.

El controlador IISAM de Exchange fallará si Microsoft Exchange o Microsoft Outlook aún no están instalados

El controlador IISAM de Exchange necesita que Microsoft Exchange o Microsoft Outlook estén totalmente instalados antes de realizar la instalación de Visual Studio. Si ninguno de los dos está instalado, el controlador IISAM de Exchange no se registrará automáticamente.

Para solucionar el problema después de instalar Visual Studio 6.0, puede instalar Microsoft Exchange o Microsoft Outlook y después registrar manualmente el controlador IISAM de Exchange mediante esta línea de comandos:

Regsrv32 msexch35.dll

    Data Access SDK 2.0

    Esta sección trata todos los problemas relacionados con Data Access SDK 2.0.

    Requisitos de la instalación

    El Data Access SDK, incluido con Visual Studio necesita que estén instalados los componentes de acceso a datos de Microsoft antes de instalar este SDK. De no ser así, cuando se ejecute la instalación del SDK, aparecerá el mensaje:

    ODBC no está instalado en este equipo. Si continúa, recibirá mensajes de error. ¿Desea continuar?

    Para resolverlo, asegúrese de que ha ejecutado la instalación de Visual Studio antes de instalar el Data Access SDK.

    Cómo instalar Microsoft Data Access SDK 2.0

    Microsoft Data Access SDK 2.0 puede instalarse ejecutando el archivo DASDK.exe desde el directorio \dasdk del disco 2.

    El SDK incluye ejemplos de Microsoft Data Access Components y más información y herramientas para los programadores que utilicen ODBC y OLE DB.

    Antes de usar SQL Server 6.5 o anterior con MDAC 2.0

    Debe actualizar los procedimientos de catálogo si piensa utilizar SQL Server 6.5 o posterior con MDAC 2.0. Consulte el archivo de Ayuda del controlador de SQL Server, drvssrvr.hlp, para obtener más información.

    • En sistemas NT: \winnt\system32
