Indizar un sitio Web recién creado

Cuando crea un nuevo sitio Web mediante Internet Information Server (IIS), el sitio no se indiza de forma predeterminada y por lo tanto no puede buscar documentos en ese sitio con Index Server.

Para indizar un nuevo sitio Web
  1. En el panel izquierdo de Microsoft Management Console (MMC), haga clic con el botón secundario en el sitio Web recién creado.
  2. Haga clic en Propiedades en el menú.
  3. En la hoja de propiedades Directorio particular, seleccione Indizar este directorio.
  4. Haga clic en Aceptar.

A continuación, debe agregar un catálogo para el sitio nuevo. Para aprender cómo agregar un catálogo, vea Crear catálogos en la documentación principal.

Ejecutar Webhits fuera del proceso de IIS

Los formularios de consulta de ejemplo de Index Server sample configuran la aplicación ISAPI Webhits para ejecutarse fuera del proceso de IIS, ya que las aplicaciones ISAPI están generalmente configuradas para ejecutarse fuera de proceso cuando no son confiables. Es mejor ejecutar las aplicaciones no confiables fuera de proceso para que así IIS no se vea afectado si la aplicación falla. Webhits está configurado en los ejemplos de Index Server para ejecutarse fuera de proceso ya que utiliza filtros de texto (IFilter) para mostrar los aciertos de las consultas en archivos. Al provenir los filtros de texto de varias compañías, es difícil garantizar que todos sean robustos.

Los filtros incluidos con Index Server han sido minuciosamente comprobados. Éstos incluyen los filtros de HTML, Microsoft Office y de texto. Si no ha instalado ningún filtro de terceros es seguro (y más eficiente) ejecutar los Webhits en el proceso IIS en su servidor Web.

El Administrador de servicios Internet de IIS puede configurar los directorios virtuales para ejecutarse en el proceso de IIS o fuera de él. Para configurar los Webhits para ejecutarse en el proceso o fuera de él, modifique las propiedades del directorio virtual en el que residen los archivos .htw. A continuación, en la hoja de propiedades Directorio, establezca adecuadamente la casilla de verificación Ejecutar en espacio de memoria separado (proceso aislado). Index Server instala de forma predeterminada ejemplos de archivos .htw en el directorio virtual /IISsamples/ISsamples/Oop.

Si no puede ejecutar Webhits en el proceso IIS, las limitaciones en IIS imponen restricciones a los Webhits. Por ejemplo, los archivos en directorios virtuales en equipos que no sean el servidor Web no se podrán mostrar mediante Webhits cuando Webhits se ejecuta fuera del proceso IIS. Además, los archivos con extensiones no asociadas a un IFilter no se pueden mostrar con Webhits cuando Webhits se ejecuta fuera del proceso IIS y cuando el parámetro del Registro WebhitsDisplayScript no está establecido como 2.

Puede solucionar estas limitaciones, pero la solución compromete la seguridad del servidor de Web. Utilice esta solución sólo si la seguridad no es una preocupación significativa en la instalación. La solución requiere que el administrador realice las siguientes tareas:

  • Permitir acceso anónimo al directorio mediante el Administrador de servicios Internet.
  • Configurar una cuenta que tenga acceso administrativo como usuario predeterminado del directorio.
Para configurar una cuenta de usuario
  1. Haga clic en Inicio, seleccione Windows NT 4.0 Option Pack, Microsoft Transaction Server y haga clic en Transaction Server Explorer.
  2. En MMC, haga doble clic en la carpeta Microsoft Transaction Server.
  3. Abra la carpeta Equipos.
  4. Abra Mi PC (o la carpeta que representa el servidor Web).
  5. Abra Paquetes instalados .
  6. Haga clic con el boton secundario del mouse (ratón) en el paquete que representa Iissamples/Issamples/Oop y haga clic en Propiedades en el menú para abrir la hoja de propiedades.
  7. En la hoja de propiedades Identidad, escriba un nombre de usuario y una contraseña en los cuadro Usuarior y Contraseña y haga clic en Aceptar.

    8. Esta cuenta debe tener privilegios administrativos en el equipo local.
