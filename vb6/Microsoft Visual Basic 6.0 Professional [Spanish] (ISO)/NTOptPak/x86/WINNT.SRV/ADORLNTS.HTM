Descripción del producto

Microsoft ActiveX Data Objects (ADO) es una interfaz basada en Automatización para obtener acceso a datos. ADO utiliza la interfaz de OLE DB para tener acceso a un amplio conjunto de orígenes de datos, incluyendo pero no limitándose a los datos proporcionados mediante ODBC.

Los usuarios de RDO y DAO deberán sentirse rápidamente cómodos con la programación en ADO, ya que el diseño general de ADO proviene de nuestra experiencia en el desarrollo de estas interfaces.

Nuevas características

Esta versión de ADO incorpora varias características para proporcionar al programador más eficiencia y control sobre sus aplicaciones. (Consulte la documentación para obtener más información acerca de cómo utilizar estas características.) Entre las nuevas características se incluye las siguientes:

  • Integración con Microsoft Client Cursors

    • Como en RDO 2.0, los programadores ahora tienen un motor de cursor de alto rendimiento en el cliente que acepta actualización por lotes optimista, objetos Recordset sin conexión y mucho más.

  • Comandos como métodos de conexión

    • Con esta característica, los comandos asociados con esta conexión se convierten en métodos en esta interfaz de conexión.

  • Datos remotos

    • Con esta característica, los usuarios de ADO pueden transmitir datos a través de HTTP a un cliente, trabajar con dichos datos devolverlos al servidor HTTP de nuevo.

Notas de la instalación

Antes de ejecutar el programa de instalación

Antes de instalar ADO compruebe que sus equipos cliente y servidor cumplen los requisitos mínimos de hardware y software.

Requisitos de hardware

Para instalar ADO debe cumplir ciertos requisitos de hardware, que incluyen los siguientes:

  • PC compatible x86 con Microsoft® Windows® 95 o Microsoft® Windows NT®, o una máquina con un procesador basado en ALPHA de Digital Equipment Corporation.
  • Un disco duro con un mínimo de 14 megabytes (MB) de espacio disponible para la instalación completa de todos los componentes actualizados necesarios para la ejecución de ADO.
  • Un mínimo de 12 MB de RAM en los equipos cliente.
  • Un mínimo de 24 MB de RAM en los equipos servidor (se recomiendan 32 MB de RAM).

Requisitos de software

Antes de instalar ADO, su equipo debe cumplir los siguientes requisitos de software:

  • Cualquiera de las siguientes plataformas: Windows 95, Windows NT Workstation 4.0 o posterior, o Windows NT Server 4.0 (con Service Pack 3) o posterior.

Para utilizar ADO necesitará un entorno de programación que acepte objetos de Automatización. Actualmente, esto incluye (pero no se limita a ello) productos como Microsoft® Internet Information Server 3.0 ó 4.0 (con páginas Active Server), los productos de Microsoft Office y Microsoft Visual Studio™.

Problemas conocidos

  • Cuando utilice el método Open del objeto Connection de ADO en Microsoft® Visual C++® o Microsoft® Visual J++™, u otros lenguajes que requieran especificar todos los parámetros opcionales, necesitará especificar un cuarto parámetro con el valor -1. Este parámetro es un marcador de posición para una opción futura. Los usuarios de Microsoft® Visual Basic® no necesitan establecer este parámetro, ya que tendrá, de forma predeterminada, el valor correcto.


  • Si está ejecutando una consulta JOIN con una base de datos Oracle y solicita un cursor keyset o dinámico, obtendrá un error en vez de un cursor de baja categoría. Oracle sólo acepta cursores estáticos de sólo lectura con las combinaciones.


  • La ejecución de dos o más objetos Command en la misma conexión a una base de datos de Microsoft® SQL Server provocará un error si alguno de los objetos Command es un procedimiento almacenado con parámetros de salida. Si está ejecutando procedimientos almacenados de Microsoft SQL Server con parámetros de salida, deberá utilizar conexiones distintas para cada objeto Command o desconectar los demás objetos Command de la conexión (por ejemplo, en Microsoft Visual Basic, Set cmd.ActiveConnection = Nothing).


  • Cuando utilice la cláusula ORDER BY en una consulta con Microsoft SQL Server que tenga instalado Service Pack 3 de Microsoft SQL Server, las columnas a las que haga referencia la cláusula ORDER BY deben especificarse también en la cláusula SELECT.


  • Cuando abra un Recordset cuya propiedad CursorLocation tenga el valor adUseClient (por ejemplo, un cursor del cliente), la propiedad UnderlyingValue no estará disponible en los objetos Field del Recordset resultante.


  • En ADO 1.5, es posible copiar valores de un campo Recordset a otro mediante una línea de comandos como ésta:
    destRst.Fields("micampo") = srcRst.Fields("micampo")
    Se ha comprobado que especificar explícitamente .Value en ambos lados de la expresión mejorará el rendimiento y evitará un problema conocido con una pequeña fuga de memoria en esta área. La nueva sintaxis que hay que utilizar sería: 
    destRst.Fields("micampo").Value  = srcRst.Fields("micampo").Value
    Esto debe aumentar el rendimiento y evitar la fuga de memoria.
