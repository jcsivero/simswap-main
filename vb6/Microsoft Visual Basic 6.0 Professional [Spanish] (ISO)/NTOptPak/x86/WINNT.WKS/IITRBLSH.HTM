Obtener asistencia mientras trabaja

Existen grupos de noticias de igual a igual para ayudarle a interactuar con otros usuarios de nuestros productos, incluidos Microsoft Most Valuable Professionals (MVP). Puede utilizar cualquier software lector de noticias para tener acceso a estos grupos de noticias; sin embargo, le sugerimos que utilice Internet Mail and News. Independientemente del lector de noticias o el cliente de noticias que utilice, es posible que tenga que configurarlo para que lea los grupos de noticias. Cuando se le pida el servidor de noticias, especifique msnews.microsoft.com. No necesita escribir un nombre de cuenta o una contraseña. Antes de exponer en los grupos de noticias, revise las reglas de conducta de los grupos de noticias de Microsoft. Para obtener más información acerca de los grupos de noticias de Microsoft vea http://www.microsoft.com/support/news/ y elija Internet Information Server.

Instalación

Si tiene problemas al instalar PWS o con el funcionamiento del programa tras la instalación, compruebe lo siguiente.

Desinstalar versiones anteriores

PWS 4.0 no actualiza automáticamente PWS 4.0 Alpha, Beta 1 o Beta 2. Si tiene PWS 4.0 Alpha, Beta 1 o Beta 2, debe desinstalarlo antes de instalar PWS 4.0. Puede hacerlo utilizando la aplicación Agregar o quitar programas del Panel de control o mediante el programa de instalación de PWS

  1. Haga clic en Inicio,seleccione Programas, Microsoft Internet Information Server y haga clic en Instalación de Internet Information Server.
  2. Seleccione Quitar todo.

Probar la instalación

Tras instalar, puede probar su instalación mediante Internet Explorer para ver los archivos de su directorio particular.

 

Para probar un sitio Web en la Intranet
  1. Asegúrese de que el equipo tiene una conexión de red activa y de que el servicio de servidor WINS (u otro método de resolución de nombres) funciona.
  2. Inicie un explorador de Web, como Microsoft Internet Explorer.
  3. Escriba el localizador de recursos universal (dirección URL) del directorio particular del nuevo servidor.

    4. La dirección URL es“http://” seguida del nombre de red de Windows de su servidor, seguido a su vez de la ruta de acceso al archivo que desea ver. (Observe las barras diagonales.) Por ejemplo, si su sitio Web está registrado con el servidor WINS como “Admin1” y desea ver el archivo PagPrinc.htm en el raíz del directorio particular, escriba http://admin1/PagPrinc.htmen el cuadro Dirección y, a continuación, presione ENTRAR. En la pantalla aparece la página principal.

 

Para probar un sitio Web en un equipo no conectado a una red
  1. Inicie un explorador de Web, como Microsoft Internet Explorer.
  2. Escriba el localizador de recursos universal (dirección URL) del directorio particular del nuevo servidor.

    3. La dirección URL es“http://” seguida del nombre del equipo, seguido a su vez de la ruta de acceso al archivo que desea ver. (Observe las barras diagonales.) Para buscar el nombre del equipo, haga clic en Inicio, seleccione Configuración y haga clic en Panel de control. Abra la aplicación Red para ver el nombre del equipo.

  3. Si el explorador muestra un mensaje de error que indica que no se puede encontrar el sitio o el servidor, es posible que necesite instalar Microsoft Loopback Adapter y modificar su archivo HOSTS.
    1. En la aplicación Red, seleccione la ficha Adaptadores.
    2. En la hoja de propiedades Adaptadores, haga clic en Agregar.
    3. Seleccione MS Loopback Adapter y haga clic en Aceptar. Se le pedirán los discos de instalación de Windows NT.

Derechos de usuario

Si un usuario tiene problemas para ver sus archivos publicados (es decir, el usuario recibe el error de HTTP "403; Prohibido" o errores de HTTP similares al intentar solicitar una página Web), probablemente haya un problema relacionado con los derechos de usuario configurados en el sitio Web. Con el fin de dar a los usuarios acceso a los archivos publicados, compruebe los siguientes elementos.

  1. Busque el archivo, el directorio o el directorio virtual específico al que el usuario no puede tener acceso en el Administrador de servicios Internet.
  2. Vea las propiedades de ese archivo, directorio o directorio virtual.
  3. Seleccione la hoja de propiedades Archivo (si está viendo las propiedades de un archivo) o la hoja de propiedades Directorio (si está viendo las propiedades de un directorio o directorio virtual). Asegúrese de que el directorio o el archivo tiene permisos de acceso de Lectura (la casilla de verificación Lectura está activada).
  4. Seleccione la hoja de propiedades Seguridad de archivos o Seguridad de directorios.
  5. Asegúrese de que el usuario tiene acceso anónimo, permisos de autenticación básica o permisos de Desafío/Respuesta de Windows NT que le permitirán ver el contenido: haga clic en el botón Modificar en el campo Control de autenticación y acceso anónimo.
  6. Haga clic en el botón Modificar del campo Restricciones de TCP/IP y nombre de dominiopara asegurarse de que al equipo, grupo de equipos o nombre de dominio del cliente no se le restringió el acceso al recurso.

Autenticación anónima

Si un usuario anónimo no puede tener acceso al sitio, compruebe lo siguiente.

Asegúrese de que la contraseña del usuario anónimo es la misma en el Administrador de servicios Internet (ISM) y en el Administrador de usuarios:

  1. En ISM, seleccione el sitio Web, el directorio virtual, el directorio o el archivo en el que necesita comprobar el acceso anónimo y vea sus propiedades.
  2. Seleccione la hoja de propiedades Seguridad de archivos (si está viendo las propiedades de un archivo) o la hoja de propiedades Seguridad de directorios (si está viendo las propiedades de un sitio Web, directorio o directorio virtual) y haga clic en el botón Modificar del campo Control de autenticación y acceso anónimo.
  3. En el cuadro de diálogo Métodos de autenticación, asegúrese de que la casilla de verificación Permitir acceso anónimo está activada.
  4. Haga clic en el botón Modificar para ver el nombre de usuario y la contraseña que se utilizan cuando un usuario anónimo se conecta a su sitio. Para que el sitio Web pueda sincronizar automáticamente su configuración de contraseña anónima con la establecida en Windows NT, active la casilla de verificación Habilitar sincronización automática de contraseñas (está activada de forma predeterminada para todos los recursos que admiten acceso anónimo).

Asegúrese de que el usuario anónimo tiene derechos de Inicio de sesión local en el Administrador de usuarios para dominios de Windows NT

  1. Inicie el Administrador de usuarios desde la barra de herramientas de Microsoft Management Console (MMC).
  2. En el Administrador de usuarios, seleccione Derechos de usuario en el menú Directivas.
  3. Elija el derecho Inicio de sesión local y asegúrese de que IUSR_ (o la cuenta anónima) aparece en la lista.

Si tiene problemas para evitar que un usuario anónimo tenga acceso al sitio, compruebe lo siguiente:

  1. Seleccione la hoja de propiedades Seguridad de archivos (si está viendo las propiedades de un archivo) o la hoja de propiedades Seguridad de directorios (si está viendo las propiedades de un sitio Web, directorio o directorio virtual) y haga clic en el botón Modificar del campo Control de autenticación y acceso anónimo.
  2. En el cuadro de diálogo Métodos de autenticación, asegúrese de que la casilla de verificación Permitir acceso anónimo noestá activada.

Autenticación básica

Si los usuarios con derechos de Autenticación básica tienen problemas para obtener acceso a su sitio, realice las siguientes comprobaciones.

Asegúrese de que el usuario que inicia la sesión tiene derechos de Inicio de sesión local en el Administrador de usuarios para dominios de Windows NT

  1. Inicie el Administrador de usuarios desde la barra de herramientas de Microsoft Management Console (MMC).
  2. En el Administrador de usuarios, seleccione Derechos de usuario en el menú Directivas.
  3. Elija el derecho Inicio de sesión local y asegúrese de que IUSR_ (o la cuenta anónima) aparece en la lista.

Asegúrese de que especifica un dominio de conexión predeterminado para el usuario: en el cuadro de diálogo Métodos de autenticación, haga clic en Modificar del campo Autenticación básica y escriba el nombre del dominio.

Si le preocupa la seguridad en la transmisión de contraseñas en texto simple (un estándar que se aplica a la Autenticación básica), utilice Secure Sockets Layer (SSL) para asegurar las contraseñas de texto simple. Puede configurar la autenticación de clientes mediante SSL si abre el cuadro de diálogo Comunicaciones seguras desde la hoja de propiedades Seguridad de directorios (o Seguridad de archivos). Utilice el Administrador de claves para crear peticiones de claves y el cuadro de diálogo Comunicaciones seguras para habilitar un sistema de autenticación mediante SSL.
