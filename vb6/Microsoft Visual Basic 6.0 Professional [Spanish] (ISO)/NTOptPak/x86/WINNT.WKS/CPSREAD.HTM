Antes de instalar este producto, revise este documento en su totalidad y lea el Manual del administrador de servicios de punto de conexión de Microsoft y Referencia de administración de servicios de punto de conexión de Microsoft. Estos documentos contienen información crítica para asegurar la instalación y uso correctos del producto.

Si está ejecutando Windows NT Workstation con el componente Personal Web Manager de Windows NT 4.0 Option Pack, no estará disponible toda la documentación desde el acceso directo Documentación del producto del menú Inicio. Estos significa que la documentación de Servicios de punto de conexión no estará disponible. Para ver esta documentación, tiene que verla desde Administrador de servicios Internet. Para ello:

  1. Haga clic en Inicio, Windows NT 4.0 Option Pack, Microsoft Personal Web Server, Administrador de servicios Internet.
  2. Seleccione Internet Information Server en el panel izquierdo.
  3. En el menú Ayuda, haga clic en Ayuda acerca de Internet Information Server. Aparecerá toda la documentación, incluyendo la de Servicios de punto de conexión.
Para cambiar el acceso directo Documentación del producto en el menú Inicio de manera que se muestre toda la documentación, desinstale el componente Personal Web Manager de Personal Web Server.

Problemas conocidos de los Servicios de punto de conexión

  • La primera vez que inicie el Administrador de libretas de teléfonos no podrá exponer en el Servicio de libretas de teléfonos por motivos de seguridad. Para poder exponer:
    1. Asegúrese de que el servicio FTP predeterminado está instalado y en ejecución en el equipo donde se ejecuta el Servicio de libretas de teléfonos. Si no está instalado, consulte Guía de instalación y diseño de Internet Information Server para ver los procedimientos de instalación.
    2. Compruebe que ha establecido el directorio raíz virtual para exponer. Para ello, en el panel izquierdo de Microsoft Management Console, haga doble clic en el sitio Default FTP. Debe aparecer en el árbol un directorio virtual llamado PBSData.

      Si instaló el servicio FTP después de instalar el Servicio de libretas de teléfonos, tendrá que configurar el directorio virtual PBSData.

      Para ello, haga clic con el botón secundario del mouse (ratón) en el sitio Default FTP, seleccione Nuevo y haga clic en Directorio virtual. Después, siga las instrucciones del Asistente para nuevo directorio virual:
      1. Cuando se le pida el alias del directorio virtual, escriba PBSdata.
      2. Cuando se le pida la ruta física del directorio, escriba:
        c:\inetpub\wwwroot\Phone Book Service\data
      3. Cuando se le pregunte si desea proporcionar permisos de acceso, asegúrese de que está activada la opción Permitir acceso de escritura y haga clic en Finalizar.
    3. Si no lo ha hecho todavía en el paso anterior, active el acceso de escritura para el directorio virtual de publicación. Para ello, haga clic con el botón secundario del mouse en PBSData y haga clic en Propiedades. En la ficha Directorio virtual, compruebe que está activada la casilla de verificación Escritura.
    4. Si desea compartir de forma segura el directorio del Servicio de libretas de teléfonos, puede especificar una cuenta de usuario de Windows NT que debe emplear el Administrador de libretas de teléfonos. Para obtener más información, consulte la sección "Configuración de seguridad" de Referencia de administración de Servicios de punto de conexión.
    Para obtener más información, consulte el Manual de administración de Servicios de punto de conexión.
  • Si desinstala el Servicio de libretas de teléfonos, los directorios virtuales no se quitarán automáticamente de Microsoft Management Console; tendrá que quitarlos manualmente.
  • Si está ejecutando de forma remota los Servicios de punto de conexión y se pierde la conexión con el servidor de archivos, se puede producir un error de tiempo de ejecución o se puede bloquear el Administrador de libretas de teléfonos.
  • El Administrador de libretas de teléfonos no acepta la exposición o importación remota de libretas de teléfonos de un tipo de plataforma a otro tipo diferente (por ejemplo, desde un equipo x86 a un equipo Alpha).
  • Los archivos de libreta de teléfonos deben ser de texto sin formato (.txt). Los archivos de libreta de teléfonos guardados con otros formatos (por ejemplo, Word, Wordpad o Rich-Text Format) no funcionarán correctamente.
  • Los archivos de libreta de teléfonos no se expondrán correctamente si los nombres de usuario o los nombres de host están escritos con un juego de caracteres de doble byte (DBCS). Exponga mediante la dirección IP del servidor de libretas de teléfonos y una cuenta de usuario SBCS.
  • Si desinstala Microsoft® Access 97 o Visual Basic® de un equipo que ejecute el Administrador de libretas de teléfonos, deberá volver a instalarlo una vez completada la desinstalación.
  • El Administrador de libretas de teléfonos sólo puede importar archivos de regiones que contengan 200 entradas de regiones como máximo mediante la interfaz gráfica. Si su archivo de regiones contiene más de 200 entradas, divídalo en dos archivos independientes o importe el archivo de regiones mediante la interfaz de línea de comandos.
