Problemas conocidos del Kit de administración del Administrador de conexión
  • En esta versión no se aceptan las características de Windows 3.1x para el Administrador de conexión. En una versión futura habrá disponible una versión del Administrador de conexión para Windows 3.1x.
  • Los perfiles de servicio creados con la versión beta 2 o anterior del asistente no son compatibles con esta versión del asistente. No puede modificar perfiles con el asistente ni crear nuevos perfiles basados en dichos perfiles.
  • No se aceptan nombres largos de archivo.
  • El perfil del servicio no puede instalar el Administrador de conexión en un subdirectorio cuya ruta de acceso tenga más de 256 caracteres.
  • Si necesita ejecutar un programa inmediatamente después de que sus usuarios ejecuten su ejecutable de perfil del servicio (por ejemplo, si tiene un archivo léame que desea mostrar automáticamente o si ha incluido un programa en el perfil del servicio que copia la configuración de Acceso telefónico a redes a su archivo .cmp para preservar la configuración de conexión), esta funcionalidad está ahora disponible.

    Para ello, cree el perfil del servicio y agregue el nombre del programa que desee ejecutar, así como las opciones o conmutadores de línea de comandos, a la sección [RunPostSetupCommandsSection] del archivo .inf asociado al perfil del servicio. (Si desea aplicar este cambio a todos los demás perfiles de servicio que cree, haga el mismo cambio en el archivo template.inf de \Archivos de programa\CMAK\cm32\enu.)

    Por ejemplo, si el programa es Miprograma.exe, la sección [RunPostSetupCommandsSection] tendría esta apariencia:

    [RunPostSetupCommandsSection]
    Miprograma.exe /opciones? /conmutadores

  • Cuando cree perfiles habilitados para PPTP o que no reconozcan el explorador en un equipo con 32 MB de RAM o menos, el asistente puede funcionar extraordinariamente despacio o incluso parecer que se ha detenido. Para mejorar el rendimiento, pruebe a crear el perfil en una máquina con más de 32 MB de RAM.
  • Cuando cree un perfil del servicio mediante el Kit de administración del Administrador de conexión, no utilice caracteres escritos con un juego de caracteres de doble byte (DBCS). Para escribir caracteres en una versión DBCS de Windows 95 o Windows NT, utilice un Input Method Editor (IME, Editor de método de entrada) configurado para producir números del juego de caracteres de byte único (SBCS). El japonés y el hangeul son algunos ejemplos de versiones de DBCS.
  • Por el momento, Microsoft Dial-Up Networking 1.2 (y, por tanto, la funcionalidad PPTP para Windows 95) sólo está disponible en inglés. Los perfiles de servicio de otros idiomas no pueden incluir la actualización de Microsoft Dial-Up Networking 1.2.

 

Problemas conocidos del Administrador de conexión
  • El Administrador de conexión no acepta los parámetros IdleThreshold e IdleTimeout en Windows NT. Por tanto, puede que no funcione correctamente el valor "Desconectar si está inactivo más de" en el Administrador de conexión.
  • El Administrador de conexión requiere ciertos archivos suministrados por Internet Explorer 3.0 o posterior. Si quita Internet Explorer, el Administrador de conexión dejará de funcionar.
  • No instale dos instancias del Administrador de conexión de Microsoft en dos carpetas diferentes del mismo equipo. Esto hará que se anule la asociación entre los perfiles de servicio originales y el Administrador de conexión. Para evitarlo, instale ambas versiones en la misma carpeta.
  • En Windows 95, si intenta establecer una conexión de Acceso telefónico a redes con su ISP, puede ver un mensaje de error en el cuadro de diálogo del Administrador de conexión de Microsoft que indique que necesita escribir un número de teléfono. Si esto sucede, haga clic en Configuración. Cuando aparezca el cuadro de diálogo Información de ubicación, siga las instrucciones y, después, haga clic en Aceptar.
  • En Windows NT, si intenta establecer más de una conexión, puede que vea el mensaje, "El módem no está configurado correctamente...". Esto sucede algunas veces si intenta marcar demasiado deprisa o si inicia el Administrador de conexión de Microsoft cuando ya se ha establecido una conexión mediante Acceso telefónico a redes. No tenga en cuenta el mensaje. Utilice una conexión cada vez.
  • En Windows NT, podría tener problemas con Servicios de acceso remoto (RAS), instalado automáticamente por el perfil de servicio del Administrador de conexión. Compruebe que RAS está configurado para utilizar un tipo de inicio manual y que tiene privilegios para iniciar RAS. Para ello,
    1. Haga clic en Inicio, Configuración, Panel de control y doble clic en Servicios.
    2. En la lista Servicios, haga doble clic en Servicio de acceso remoto.
    3. En el cuadro de diálogo Servicios, haga clic en Manual y, después, haga clic en Aceptar.
  • En Windows NT, si cambia el valor "Desconectar si está inactivo más de", la modificación no surtirá efecto hasta que salga del Administrador de conexión de Microsoft y lo reinicie.
  • Si escribe o modifica el número de teléfono en la ficha Marcando del cuadro de diálogo Configuración de conexión, compruebe que utiliza el formato adecuado en el número de teléfono. De lo contrario, el Administrador de conexión no utilizará las Propiedades de marcado que ha configurado para marcar el número de teléfono.

    El formato correcto es el siguiente:
    +[código de país] (código de área) número de teléfono
    Por ejemplo, si su código de país es 1, su código de área es 800 y su número de teléfono es 555-1212, debe escribir el número de teléfono como se indica a continuación, comprobando que incluye el signo '+' , los espacios en blanco y los paréntesis:
    +1 (800) 5551212
    Si no utiliza este formato, asegúrese de que escribe el número tal y como debe marcarse.
  • Si está utilizando Acceso telefónico a redes 1.2 de Microsoft y aparece el mensaje siguiente, haga caso omiso de él:
    Este cliente DHCP no pudo obtener una dirección IP de red del servidor DHCP. ¿Desea ver los mensajes DHCP que se produzcan?
  • El Administrador de conexión de Microsoft no funciona en Windows NT 4.0 con algunos adaptadores ISDN internos de red, incluyendo el módem ISDN Diamond Supra NetCommander.
  • El Administrador de conexión permite el uso de ISDN (RDSI) de canal único. Puede configurar manualmente el Administrador de conexión para utilizar ISDN de dos canales.

    Si está ejecutando Windows 95, siga el procedimiento descrito a continuación para configurar ISDN de dos canales:
    1. Haga clic en Inicio, Programas, Accesorios, Acceso telefónico a redes.
    2. Haga clic con el botón secundario del mouse (ratón) en el icono de la conexión para su ISP y, después, haga clic en Propiedades.
    3. Haga clic en la ficha Enlace multiple.
    4. Haga clic en Usar dispositivos adicionales y, después, haga clic en Agregar.
    5. En el cuadro de diálogo Modificar dispositivo adicional, seleccione el dispositivo para el canal ISDN adicional y escriba el número de teléfono de dicho dispositivo.

    Si desea utilizar dos canales para el icono de la conexión de reserva, repita los pasos anteriores para dicho icono.

    Cuando haya utilizado esta técnica para configurar un icono de conexión de dos canales para usarlo con el Administrador de conexión, si cambia el número de teléfono principal o el secundario, también tendrá que modificar el número de teléfono multivínculo que escribió en el paso 5.

    Si está ejecutando Windows NT, consulte la documentación de Windows NT para obtener información acerca de la configuración de conexiones ISDN de dos canales.
  • El Administrador de conexión instalará automáticamente el protocolo TCP/IP, pero no otros protocolos. Si utiliza un icono de conexión que requiera un protocolo distinto a TCP/IP, compruebe que lo instala antes de intentar conectar con su ISP. Para ello:
    1. Haga clic en Inicio, Configuración, Panel de control y, después, haga doble clic en Red.
    2. Haga clic en la ficha Servicios, Servicio de acceso remoto, Propiedades.
    3. Haga clic en el módem o en el adaptador VPN y, después, haga clic en Red.
    4. En la lista de protocolos de conexión, haga clic en el protocolo que utilice su perfil de servicio.
  • Si está ejecutando Windows NT y utiliza el Administrador de conexión para establecer una conexión de Acceso telefónico a redes, el Administrador de conexión no permite que emerja una ventana de terminal después de marcar. (No obstante, sí se permite que aparezca una ventana de terminal antes de marcar.) Si su ISP requiere que su conexión haga que emerja una ventana de terminal tras realizar la conexión, póngase en contacto con su ISP y pídale una secuencia de comandos de marcado que cree la ventana de terminal.
  • Los usuarios de Windows 95 pueden tener problemas al ver los datos en pantalla que muestra el Administrador de conexión, como resultado de la instalación de un perfil de servicio que utiliza un idioma distinto del configurado en el sistema operativo.
  • No se aceptan los números escritos con un juego de caracteres de doble byte (DBCS). Para escribir números en una versión DBCS de Windows 95 o Windows NT, utilice un Input Method Editor (IME, Editor de método de entrada) para producir números del juego de caracteres de byte único (SBCS). El japonés y el hangeul son algunos ejemplos de versiones de DBCS.
