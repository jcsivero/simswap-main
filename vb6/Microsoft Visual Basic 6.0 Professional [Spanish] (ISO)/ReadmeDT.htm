Problemas conocidos

Se sabe que existen los siguientes problemas en esta versión de Visual Database Tools. Para ver información acerca de problemas específicos de productos, vea el archivo Léame del producto correspondiente.

Sólo se admiten los controladores MS ODBC y OLE DB más recientes

Aunque los controladores más antiguos y de otros fabricantes pueden funcionar con Visual Database Tools, estas herramientas sólo están certificadas para los controladores Microsoft ODBC y OLE DB más recientes. Si se produce algún comportamiento inesperado, compruebe que está utilizando los controladores correctos.

Se muestran datos incorrectos cuando se conecta a una base de datos internacional

Si al conectarse a una base de datos internacional se muestran caracteres incorrectos o no válidos, asegúrese de que la configuración regional de su equipo coincide con la de dichos datos.

Se requiere SQL Server Service Pack 3

Para utilizar Visual Database Tools con SQL Server 6.5 se requiere SQL Server Service Pack 3. Antes de utilizar Visual Database Tools, asegúrese de que se ha instalado en el servidor Service Pack 3 o posterior.

Los siguientes problemas se refieren a Microsoft® Visual Basic®

Ejecutar consultas de "creación de tabla" en bases de datos de Oracle

Las consultas de "creación de tabla" que se ejecutan en una base de datos de Oracle mediante el proveedor Microsoft OLE DB para Oracle no funcionan correctamente. En su lugar, utilice el controlador ODBC de Microsoft para crear una conexión a la base de datos.

No se muestran todos los paneles Consulta de manera predeterminada

Al abrir una tabla en Vista de datos, en realidad está abriendo el Diseñador de consultas con un comando "SELECT * FROM table". Para abrir los paneles Diagrama, Cuadrícula y SQL en Visual Basic, en el menú Ver, haga clic en Mostrar paneles. De esta forma podrá elegir qué paneles desea abrir y cerrar.
